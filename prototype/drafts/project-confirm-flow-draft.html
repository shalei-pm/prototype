900 画布 Pad 任务界面草案

本版按小米 Pad 6 竖屏 `900 x 1440` 逻辑画布排版，并在浏览器中用 `scale(.58)` 缩略展示。整合评审预览统一使用 `559 x 872` 外框。

设计假设
  • 源画布为 `900 x 1440`，浏览器缩略展示统一为 `559 x 872` 外框。
  • 新客建档后进入项目菜单；老客识别后直接进入治疗项目确认页。
  • 项目菜单完成购买后先回到我的资产，新购项目默认选入今日治疗，再流向治疗项目确认页。
  • 这些页面都是扫码后的业务页，因此每个 Pad 画布顶部都有“登出”。
  • 支付确认使用覆盖项目菜单的半弹窗；扫码、成功和未完成使用完整任务页。
Pad 布局调整
  • 项目菜单改为顶栏轻切换、左侧分类 / 筛选、右侧项目 / 资产列表；已选摘要下沉到底部操作区。
  • 支付二维码有效期为 60 秒；成功后进入我的资产，未完成状态 3 秒后返回项目菜单。
  • 治疗项目确认页使用通栏项目卡片；AI羊点评放在内容末尾、固定按钮上方，以可变长度的自然段归纳预计时间、改善点和术前注意，不使用等宽指标表格。
  • 流程确认页平铺展示附加服务，重点标明每项预计额外耗时。
  • 底部主操作区固定，避免客户找不到下一步。
字号和触控
  • 字号以 900 画布为准：主标题 26px，模块标题 18px，项目名 16px，正文 14px，按钮 15-16px。
  • 中文字重已降档：正文 400，模块与列表标题 600-700，页面主标题 700；不再使用 800 / 900 / 950。
  • 返回、门形登出 icon、顶栏切换、筛选、列表操作、底部按钮等可点热区均按 56px 左右源画布尺寸设计，缩略展示时整体等比缩放。
  • 相邻点击元素保持 8px 以上间距，适合前台站立操作。
  • 绿和黑只用于选中态、主按钮、状态点缀，不做大面积黑底业务页。
需要确认的问题
  • 项目菜单底部已选摘要是否常驻，还是只在已有选择时出现？
  • 无项目时“安排面诊”是否直接进入服务人员确认页，还是需要先选择咨询师 / 医生类型？
  • 皮肤检测和拍照是否允许客户关闭，或由项目规则强制开启？
  • 饮品零食是否按“饮品一项 + 小食一项”分别选择？
